# 训练运行手册

## 1. 当前建议的训练起点

当前最稳妥的路线是：

- 从公开发布的 `LLaVA-Mini` checkpoint 开始
- 首次训练先以 SFT smoke run 为目标
- 优先验证“能不能跑通”，而不是“效果能不能最好”

## 2. 推荐起始模型

当前默认建议：

- 基础模型：`ICTNLP/llava-mini-llama-3.1-8b`
- 视觉塔：`openai/clip-vit-large-patch14-336`

原因：

- 代码仓和论文都直接指向这个公开模型
- 当前训练代码通过 `--model_name_or_path` 直接加载 checkpoint
- 对于你现在这条修改路线，先从已发布 checkpoint 起步最容易缩小问题范围

## 3. 论文里与当前实验最相关的细节

基于 `llava_mini_paper.md`，当前先按原论文主设定理解：

- 训练采用两阶段范式：预训练 + 指令微调
- 当前我们优先关注第二阶段，即 instruction tuning
- 原论文基础设置使用 `CLIP ViT-L/336`
- 标准压缩超参 `C=1`，即每帧压成 1 个视觉 Token
- 模态预融合层数 `N_fusion=4`
- 视频处理采用 `1 fps` 抽帧
- LLaMA-3.1-8B 变体使用图像指令数据、视频指令数据和部分开源数据，总量约 300 万样本

## 4. 当前主入口

主入口脚本：

- `scripts/train_solo_sft.sh`

环境安装脚本：

- `scripts/setup_train_env.sh`

模型下载脚本：

- `scripts/download_train_assets.sh`

## 5. 当前执行顺序

当前推荐顺序：

1. 登录 GPU worker
2. 确认 Python 环境是否可复用
3. 补齐训练依赖
4. 下载模型和视觉塔
5. 准备最小 smoke 数据
6. 启动短程 SFT
7. 记录结果和问题

## 6. 当前已知问题

截至现在，worker 上已确认：

- GPU 可见
- 但当前默认 Python 环境里还没有 `torch`
- 也没有 `transformers`、`deepspeed`、`peft`、`bitsandbytes`、`decord`

因此当前的真正 blocker 是环境，而不是训练脚本本身。
