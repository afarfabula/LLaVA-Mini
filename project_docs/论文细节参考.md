# 论文细节参考

本文档只摘录当前实验最直接相关的 `LLaVA-Mini` 论文细节，用于指导训练启动。更完整的背景可参考仓库外的 `llava_mini_paper.md`。

## 1. 与当前训练最相关的结论

- LLaVA-Mini 的关键点不是单纯减少视觉 Token，而是在 LLM 前引入模态预融合
- 视觉 Token 对 LLM 的作用主要集中在前层
- 因此可以先在 LLM 前做预融合，再极限压缩送入 LLM 的视觉 Token 数量

## 2. 原论文训练范式

论文采用两阶段训练：

1. 视觉语言预训练
2. 指令微调

对于我们当前任务，更直接相关的是第二阶段，即 instruction tuning。

## 3. 原论文默认关键设定

- 视觉编码器：`CLIP ViT-L/336px`
- 标准压缩参数：`C=1`
- 模态预融合层数：`N_fusion=4`
- 图像侧训练数据：558K 预训练数据 + 665K 指令微调数据
- 视频侧：按 `1 fps` 抽帧

## 4. LLaMA-3.1 变体细节

论文还给出了一个使用 `LLaMA-3.1-8B-Instruct` 的变体：

- 视觉塔仍为 `CLIP ViT-L/336px`
- 指令微调数据包含 665K 图像数据、100K 视频指令数据和部分开源数据
- 总体训练样本规模约为 300 万
- 训练使用 8 张 A800

## 5. 对当前实验的启发

结合你当前仓库状态和论文细节，当前最稳的策略是：

- 先加载公开发布 checkpoint
- 先在 worker 上完成一次短程 SFT smoke run
- 先验证训练链路、显存、张量形状和 checkpoint 落盘
- 在 smoke run 之后，再决定是否继续引入更激进的结构改动
