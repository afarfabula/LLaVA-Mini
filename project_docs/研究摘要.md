# 研究摘要

## 1. 核心问题

当前长视频 Token Reduction 方法虽然提升了效率，但很容易引入一个新的失败模式：

- 时间维压缩后，模型可能还能理解大致剧情
- 空间维被极度压缩后，细粒度局部信息会严重丢失

这种问题通常表现为：

- 早期帧被删掉后，多轮问答时无法回溯关键证据
- 过强的空间池化会抹平 OCR、小物体和象限级位置信息

本项目的核心问题是：

如何在保持长上下文效率的同时，尽可能保住回答细粒度问题所需的空间细节。

## 2. 研究动机

本项目受到两条研究线索启发：

- `TimeChat-Online`：长流式视频中存在大量时间冗余，可以在进入 LLM 前先做去冗余
- `LLaVA-Mini`：视觉 Token 真正关键的阶段主要集中在前层，因此有机会大幅压缩输入 LLM 的视觉 Token 数量

因此，本项目不想在“时间效率”和“空间细节”之间二选一，而是尝试把两者正交结合。

## 3. 当前主张

当前设想是一个查询驱动的时空弹性压缩路线：

- 在时间轴上做激进但可控的去冗余
- 为每帧保留低成本的粗粒度空间锚点
- 在主干序列之外保留轻量级高清细节 Buffer
- 在中层根据文本意图按需读取更细的空间信息

简化描述如下：

- 时间维：尽可能先剪枝
- 空间维：避免所有帧都一刀切压成固定极小 Token
- 中间层：让 Query 驱动更高分辨率信息的唤醒

## 4. 当前状态

截至目前：

- 研究动机已经比较清晰
- 仓库里已经做了一些基础开发
- 训练侧路径和脚本正在整理
- 还没有完成真实 GPU 级别的验证

因此当前阶段的重点不是追榜，而是做可控验证：

1. 把训练链路先整理成可复现
2. 确认当前改动能够在 GPU 上启动
3. 确认张量形状、显存和 checkpoint 落盘没有硬错误
4. 在此基础上再进入效果验证
