# 任务计划

## 1. 当前总目标

在 `LLaVA-Mini` 上构建一条稳定、可复现的训练侧工作流，用来支持后续围绕“时空弹性压缩”方向的持续迭代。

## 2. 当前真实状态

已经成立的事实：

- 研究动机已经整理
- 仓库中已存在一些针对性的基础开发
- 训练相关脚本中的绝对路径已经清理
- 已确认可通过 `mlx worker login` 进入 GPU worker
- 已确认当前 worker `3759140` 上有 2 张 H20 可用
- 已准备最小 smoke 数据

尚未成立的事实：

- 当前改动还没有完成一次真实的 GPU smoke run
- 训练环境依赖还没有装齐
- 预训练模型和视觉塔还没有下载到本地
- 还没有确认一次可落 checkpoint 的短程训练

## 3. 当前约束

- 这台机器当前只负责训练侧工作
- 评测工作不在这台机器上执行
- 当前优先级是先跑通训练，而不是扩展 benchmark
- 第一次 GPU 运行应以 smoke test 为目标，而不是效果最优

## 4. 分阶段计划

### 阶段 0：仓库与文档整理

目标：

- 把训练空间和文档入口整理清楚

状态：

- [x] 清理训练相关绝对路径
- [x] 建立统一项目文档目录
- [x] 把任务文档改为中文
- [x] 明确训练主入口脚本

### 阶段 1：训练环境就绪

目标：

- 在 GPU worker 上形成可训练环境

检查项：

- [ ] 确认 Python 解释器与环境管理方式
- [ ] 安装 `torch`、`transformers`、`deepspeed`、`peft`、`bitsandbytes`、`decord`
- [ ] 确认 `torch.cuda.is_available()` 为 True
- [ ] 确认 `huggingface-cli` 可用

### 阶段 2：模型与数据资产到位

目标：

- 具备启动一次 SFT smoke run 的基本资产

检查项：

- [ ] 下载 `ICTNLP/llava-mini-llama-3.1-8b`
- [ ] 下载 `openai/clip-vit-large-patch14-336`
- [x] 准备最小 smoke 数据
- [ ] 确认 `DATA_PATHS` 与 `IMAGE_FOLDERS`

### 阶段 3：首个 GPU Smoke Run

目标：

- 先验证训练链路真实可跑

检查项：

- [ ] 模型可以在 GPU 上启动
- [ ] 前向和反向都能走通
- [ ] 没有立刻 OOM 或 device mismatch
- [ ] 能成功落一个 checkpoint
- [ ] 把结果写入 `实验进展.md`

## 5. 当前下一步

当前优先顺序如下：

1. 在 worker 上确认是否有现成可复用的 Python 环境
2. 复用 worker 现有 `torch`，安装缺失训练依赖
3. 确认 `huggingface-cli` 可用
4. 下载基础 checkpoint 和 vision tower
5. 用 `data/smoke/train.json` 启动首个 smoke run
